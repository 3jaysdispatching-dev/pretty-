# Accessibility

Accessibility is a foundational requirement, not an add-on.

## Color and contrast

- Maintain a minimum contrast ratio of 4.5:1 for body text and 3:1 for UI components.
- Avoid using color alone to convey meaning.

## Keyboard and focus

- Ensure every interactive element can be reached via keyboard.
- Provide visible focus states that meet contrast and size guidelines.

## Semantic markup

- Use proper HTML semantics and ARIA roles for complex components.
- Label controls clearly, and include descriptive screen reader text when needed.

## Motion and animation

- Respect user preferences for reduced motion.
- Use motion to support meaning and not to distract.
