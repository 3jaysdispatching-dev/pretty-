# Usage Guidelines

This section explains how to choose patterns, when to reuse components, and how to adapt the system to new product needs.

## Component selection

- Select the simplest component that meets the user need.
- Prefer a standard pattern over a custom one.
- Use the button system for actions, cards for groups of related content, and form controls for input tasks.

## Dos and don'ts

- Do use the design tokens for color, spacing, and typography.
- Do keep interaction states consistent across components.
- Don't create custom colors or spacing values without approval.
- Don't overload layouts with too many competing visual styles.

## Implementation notes

- Document edge cases and responsive behavior.
- Keep components composable and accessible by default.
- Use the code libraries as reference implementations when available.
